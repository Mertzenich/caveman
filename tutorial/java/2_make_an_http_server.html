Make an HTTP Server

  • Edit your src/example/Main.java file so it has the following contents. 

    package example;

import com.sun.net.httpserver.HttpServer;

import java.net.InetSocketAddress;
import java.util.List;

public class Main {
    public static void main(String[] args) throws Exception {
        var server = HttpServer.create(new InetSocketAddress(9999), 0);
        server.createContext("/", exchange -> {
            var headers = exchange.getResponseHeaders();
            headers.put("Content-Type", List.of("text/html"));
    
            exchange.sendResponseHeaders(200, 0);
            try (var os = exchange.getResponseBody()) {
                os.write("Hello, world".getBytes());
            }
        });
    
        server.start();
    }
}

    Thats a lot, but we can go through it.

    com.sun.net.httpserver.HttpServer is an HTTP Server that comes with Java.

    The 0 in HttpServer.create(new InetSocketAddress(9999), 0); just means "use a default value." For what isn't super important, but if you are curious its the "socket backlog". I'm sure that means something to smart people.

    server.createContext("/", exchange -> { ... }) tells the server how you want to handle requests. It will call the handler for all requests. The "/" means to use the given function to handle all requests that have a URI that starts with /. Which is all of them.

    exchange.getResponseHeaders() gives something we can mutate to send headers with the response, which is what headers.put("Content-Type", List.of("text/html")); is doing.

    Then finally exchange.sendResponseHeaders(200, 0); sends the headers off and we write the response body out into the OutputStream returned by exchange.getResponseBody(). The 0 there means "no guarentees how large the response body will be."

    Its a lot of steps, but each one individually is straight forward.

  • Run the server with just run

    You won't get any output in the terminal. This is normal.

  • Open your web browser and go to http://localhost:9999
